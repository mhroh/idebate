# iDebate 인터페이스 변경 원인 진단 준비 문서

작성일: 2026-06-16

## 화면 구성을 담당하는 코드 위치

현재 저장소 `mhroh/idebate`에서 앱 화면 구조를 결정하는 중심 파일은 `app.py`입니다.

주요 위치:

| 파일 | 위치 | 화면 역할 |
|---|---:|---|
| `app.py` | 56행 부근 `main()` | Streamlit 앱의 전체 화면 흐름을 결정합니다. |
| `app.py` | 60-62행 부근 | 서비스가 `off`일 때 휴식/중지 화면을 보여주고 종료합니다. |
| `app.py` | 67행 부근 `with st.sidebar:` | 왼쪽 사이드바 영역을 구성합니다. |
| `app.py` | 69-77행 부근 | 사이드바 제목, 대화명 입력, 대화 종료 버튼을 구성합니다. |
| `app.py` | 82-89행 부근 | 기존 대화 메시지를 채팅 형태로 출력합니다. |
| `app.py` | 91행 부근 `st.chat_input(...)` | 화면 하단의 채팅 입력창을 구성합니다. |
| `app.py` | 103-128행 부근 | 사용자 메시지와 assistant 응답을 `st.chat_message`로 표시합니다. |
| `mermaid_utils.py` | 전체 | Mermaid 차트 내용을 생성하고 렌더링합니다. |

현재 코드 기준으로 CSS, 별도 프론트엔드 템플릿, HTML 파일, Streamlit theme 설정 파일은 발견되지 않았습니다. 따라서 화면의 큰 구조는 대부분 `app.py` 안의 Streamlit 컴포넌트 배치로 결정됩니다. 2026-06-16 commit `8f10cb9` 이후 메인 챗봇 화면에서는 시각화 도구 UI가 제거된 상태입니다.

## Google Sheet 설정값이 화면에 영향을 주는 방식

설정값은 `utils/gs.py`의 `getSetupInfo()`가 읽습니다. 이 함수는 Streamlit Secrets의 `sheet_url`이 가리키는 Google Sheet에서 `"정보"` 시트 B열을 읽어 `setupInfo`로 저장합니다.

화면에 직접 또는 간접 영향을 줄 수 있는 설정값은 다음과 같습니다.

| Google Sheet 위치 | 설정 키 | 화면 영향 |
|---|---|---|
| `"정보"` 시트 B1 | `url` | 사용자별 대화 기록용 Google Sheet를 엽니다. 이 Sheet 안의 템플릿/요약 시트 구조가 대화 기록 및 종료 동작에 영향을 줄 수 있습니다. |
| `"정보"` 시트 B2 | `serviceOnOff` | `off`이면 일반 채팅 화면 대신 휴식/중지 안내 화면만 표시됩니다. |
| `"정보"` 시트 B3 | `AI` | 현재 `app.py` 화면 분기에는 직접 사용되지 않습니다. 다만 운영자가 표시용으로 기대한 값과 코드가 다를 수 있어 확인 대상입니다. |
| `"정보"` 시트 B5 | `model` | 화면 배치 자체를 바꾸지는 않지만, 응답 성공/실패와 응답 양상에 영향을 줄 수 있습니다. 오류가 나면 Streamlit 오류 메시지가 화면에 표시됩니다. |
| `"정보"` 시트 B6 | `max_tokens` | 응답 길이에 영향을 줍니다. 긴 응답은 채팅 화면의 스크롤과 표시 체감에 영향을 줄 수 있습니다. |
| `"정보"` 시트 B7 | `temperature` | 답변 스타일에 영향을 줍니다. 화면 구성 자체를 바꾸지는 않습니다. |
| `"정보"` 시트 B8 | `select` | 현재 `app.py` 화면 분기에는 직접 사용되지 않습니다. |
| `"정보"` 시트 B9 | `system` | 대화의 역할, 말투, 답변 형식에 영향을 줍니다. 사용자는 인터페이스가 바뀐 것처럼 느낄 수 있습니다. |
| `"정보"` 시트 B10 | `a_p` | 대화 종료 후 평가 생성 흐름에 쓰일 수 있는 프롬프트입니다. 현재 `end_conversation()` 초반에 `return`이 있어 실제 평가 저장 흐름은 실행되지 않는 구조로 보입니다. |
| `"정보"` 시트 B11 | `e_p` | 대화 종료 후 평어 생성 흐름에 쓰일 수 있는 프롬프트입니다. 현재는 위와 같은 이유로 실제 실행 여부 확인이 필요합니다. |
| `"정보"` 시트 B12 | `stream` | `true`이면 응답이 스트리밍으로 표시됩니다. `false`이면 현재 코드의 `message_processing()` 흐름과 맞지 않아 응답 표시 방식에 문제가 생길 수 있습니다. |

특히 화면 차이를 만들 수 있는 값은 `serviceOnOff`, `system`, `stream`, 그리고 B1의 기록용 Google Sheet 구조입니다.

## 인터페이스가 달라질 수 있는 원인 후보

코드 기준 후보는 다음과 같습니다.

1. 배포 중인 앱이 다른 Repository / Branch / Main file path를 보고 있을 수 있습니다.
   - 로컬 조사 기준 앱 진입 파일은 `app.py`입니다.
   - 실제 `idebate.streamlit.app` 설정은 Streamlit Cloud에서 확인 필요합니다.

2. `idebate.streamlit.app`의 `sheet_url`이 다른 설정용 Google Sheet를 가리킬 수 있습니다.
   - 코드 안에는 `idebate.streamlit.app`, `idebate01`, `idebate02`, `idebate03`별 분기가 없습니다.
   - 앱별 차이는 Streamlit Cloud Secrets의 `sheet_url`에서 생길 가능성이 큽니다.
   - Streamlit Cloud에서 확인 필요합니다.

3. `"정보"` 시트 B2 `serviceOnOff`가 `off`일 수 있습니다.
   - 이 경우 채팅 UI 대신 휴식/중지 안내 화면만 보입니다.
   - Google Sheet에서 확인 필요합니다.

4. 복구 전 GitHub 코드에 시각화 도구 영역이 추가되어 화면이 달라졌습니다.
   - `97d13e5`에서 `app.py` 사이드바에 시각화 도구, 테스트 차트 보기, 수동 차트 만들기 UI가 추가되었습니다.
   - 2026-06-16 commit `8f10cb9`에서 해당 UI는 메인 챗봇 화면에서 제거되었습니다.

5. Mermaid 관련 의존성이 배포에 반영되어 새 UI가 보였을 수 있습니다.
   - `requirements.txt`에 `streamlit-mermaid`가 있습니다.
   - `mermaid_utils.py`는 남아 있지만, 2026-06-16 commit `8f10cb9` 이후 `app.py` 메인 화면에서는 Mermaid 테스트/수동 차트 UI를 노출하지 않습니다.

6. `"정보"` 시트 B9 `system` 값이 바뀌어 챗봇의 응답 방식이 달라졌을 수 있습니다.
   - 버튼이나 입력창 배치는 그대로여도, 안내 문구와 답변 형식이 달라져 인터페이스 변화처럼 느껴질 수 있습니다.
   - Google Sheet에서 확인 필요합니다.

7. `"정보"` 시트 B12 `stream` 값이 바뀌어 응답 표시 체감이 달라졌을 수 있습니다.
   - `true`이면 답변이 점진적으로 표시됩니다.
   - `false`인 경우 현재 스트림 처리 코드와 맞지 않을 수 있어 표시 문제가 생길 수 있습니다.
   - Google Sheet에서 확인 필요합니다.

8. B1의 기록용 Google Sheet 구조가 달라졌을 수 있습니다.
   - `initialize()`는 B1 URL로 열린 문서에서 사용자 이름의 워크시트를 찾거나, 없으면 `템플릿` 시트를 복제하려고 합니다.
   - 기록용 Sheet의 `템플릿`, `수업요약` 시트 구조가 달라지면 대화 시작/종료 동작이 달라질 수 있습니다.
   - 연결된 Google Sheet에서 확인 필요합니다.

9. Streamlit 버전 또는 배포 환경 차이로 기본 UI 스타일이 달라질 수 있습니다.
   - `requirements.txt`는 `streamlit>=1.0`처럼 넓은 버전 범위를 사용합니다.
   - Streamlit Cloud가 설치하는 실제 버전에 따라 기본 채팅 UI, 사이드바, 버튼 스타일이 달라질 수 있습니다.
   - Streamlit Cloud 로그 또는 package 설치 로그에서 확인 필요합니다.

10. 브라우저 캐시 또는 Streamlit 앱 캐시 때문에 일시적으로 이전/새 화면이 섞여 보일 수 있습니다.
   - `getSetupInfo()`는 `@st.cache_data(ttl=300)`으로 5분 캐시됩니다.
   - 설정 변경 직후에는 즉시 반영되지 않을 수 있습니다.
   - Streamlit Cloud에서 재부팅 후 확인 필요합니다.

### 복구 전 문구가 있던 코드 위치

확인 결과, 복구 전 실제 화면에 보였던 문구들은 Google Sheet 설정값이 아니라 `app.py`에 직접 작성되어 있었습니다. 2026-06-16 commit `8f10cb9` 이후 `시각화 도구`, `테스트 차트 보기`, `수동 차트 만들기` UI는 메인 챗봇 화면에서 제거되었습니다.

| 화면 문구 | 코드 위치 | 코드 성격 |
|---|---:|---|
| `교육용 챗봇` | `app.py:69` | `st.title("교육용 챗봇")`로 하드코딩 |
| `시각화 도구` | 복구 전 `app.py:82` | `st.subheader("🎨 시각화 도구")`로 하드코딩되어 있었고, `8f10cb9`에서 제거됨 |
| `테스트 차트 보기` | 복구 전 `app.py:84` | `st.button("📊 테스트 차트 보기")`로 하드코딩되어 있었고, `8f10cb9`에서 제거됨 |

따라서 이 네 문구는 설정 시트의 B8 `select` 값에서 온 것이 아닙니다.

### B8 select 값 반영 여부

`utils/gs.py`는 `"정보"` 시트 B8 값을 읽어 `setupInfo["select"]`에 저장합니다.

코드 위치:

```text
utils/gs.py:100
temp["select"] = data[7]
```

하지만 현재 `app.py`에서는 `setupInfo["select"]`를 화면 제목, 버튼 이름, 사이드바 구성, 채팅 UI 분기에 사용하지 않습니다.

확인 결과:

| 항목 | 판단 |
|---|---|
| B8 `select` 값이 읽히는가 | 예, `utils/gs.py`에서 읽습니다. |
| B8 `select` 값이 화면 제목에 쓰이는가 | 아니오 |
| B8 `select` 값이 사이드바 문구에 쓰이는가 | 아니오 |
| B8 `select` 값이 UI 분기 조건으로 쓰이는가 | 현재 코드 기준 아니오 |
| `상대심문 연습 도우미`가 화면에 자동 반영되는가 | 현재 코드 기준 아니오 |

### 복구 전 왜 설정은 상대심문인데 화면은 시각화 도구 중심으로 보였는가

원인은 코드 기준으로 명확합니다.

1. `select` 값은 설정 시트에서 읽지만, 화면 구성에 사용되지 않습니다.
   - B8이 `상대심문 연습 도우미`여도 `app.py`는 그 값을 제목이나 메뉴에 표시하지 않습니다.

2. 복구 전 사이드바 제목과 도구 영역은 `app.py`에 고정 문구로 들어 있었습니다.
   - `교육용 챗봇`
   - `시각화 도구`
   - `테스트 차트 보기`
   - `수동 차트 만들기`

3. `97d13e5`에서 Mermaid 시각화 도구가 추가되어, 설정값과 무관하게 모든 앱 화면에 해당 영역이 보였습니다.
   - 복구 전에는 이 영역이 `app.py`의 사이드바에 항상 렌더링되었습니다.
   - 당시 코드에는 `select` 값에 따라 시각화 도구를 숨기거나 다른 화면으로 바꾸는 조건문이 없었습니다.

### A안: 시각화 도구 UI 제거하고 기존 챗봇 UI로 되돌리기

내용:

- 현재 `main` 브랜치의 최신 코드와 Claude 모델 지원은 유지합니다.
- `app.py`에서 `97d13e5`에 추가된 시각화 도구 UI 영역만 제거합니다.

장점:

- 사용자가 보는 화면을 가장 명확하게 기존 챗봇 중심 UI로 되돌릴 수 있습니다.
- 과거 commit으로 전체를 되돌리지 않기 때문에 이후 추가된 의존성/모델 지원 변경을 잃지 않습니다.
- 현재 문제의 직접 원인인 하드코딩 UI를 정리하는 방식입니다.

주의:

- `main` 브랜치를 쓰는 다른 Streamlit 앱에도 같은 화면 변경이 적용될 수 있습니다.
- 실제 수정 전에는 `idebate01/02/03`도 같은 UI를 기대하는지 확인해야 합니다.

### B안: 시각화 도구는 접힌 선택 기능으로 숨기기

내용:

- 시각화 도구를 완전히 제거하지 않고, 접힌 영역 안으로 넣거나 기본 표시를 숨깁니다.
- 필요한 운영자만 펼쳐서 쓰는 방식입니다.

장점:

- 시각화 기능을 보존할 수 있습니다.
- 향후 필요할 때 다시 노출하기 쉽습니다.

단점:

- 화면 어딘가에 시각화 도구 흔적이 남을 수 있습니다.
- “기존 챗봇 UI로 복구”라는 목표에는 A안보다 덜 깔끔합니다.

### C안: `idebate.streamlit.app`만 과거 commit 또는 별도 branch로 되돌리기

내용:

- Streamlit Cloud에서 `idebate.streamlit.app`만 시각화 도구 추가 전 코드로 배포합니다.
- 기준 후보는 `73a8473` 직후의 챗봇 UI입니다.

장점:

- 다른 앱에는 영향을 주지 않고 `idebate.streamlit.app`만 과거 화면으로 되돌릴 수 있습니다.

단점:

- 과거 commit으로 되돌리면 이후 commit의 Claude 4.5 지원, 의존성 업데이트, 캐싱 개선 등을 잃을 수 있습니다.
- Streamlit Cloud는 보통 branch 기준 배포가 자연스럽기 때문에, 과거 commit 고정 운영은 비개발자에게 관리가 어렵습니다.
- 나중에 main과 차이가 커져 유지보수가 복잡해질 수 있습니다.
