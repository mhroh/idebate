# 📸 PR 생성 완벽 가이드 (클릭 따라하기)

## 🎯 목표: GitHub에서 PR 생성 (2분 완성)

---

## Step 1: PR 생성 페이지 열기

### 📋 복사할 링크:
```
https://github.com/mhroh/idebate/compare/main...claude/analyze-unknown-task-011CUqkenKT94Az5cryNitAm
```

### 🖱️ 할 일:
1. 위 링크를 **드래그해서 복사** (Ctrl+C 또는 Cmd+C)
2. 새 탭에서 **붙여넣기** (Ctrl+V 또는 Cmd+V)
3. **엔터** 키 누르기

### ✅ 올바른 화면:

## Step 3: PR 설명 입력

### 📋 복사할 설명:
```markdown
## 📋 변경 사항

### ✅ 성능 최적화
- Google Sheets Rate Limit 자동 재시도 (exponential backoff)
- 메모리 최적화 (대화 50개 제한)
- Claude API 프롬프트 캐싱 (토큰 90% 절감)

### ✅ 안정성 강화
- 에러 복구 로직 강화
- 평가 기능 버그 수정
- 세션 관리 개선

### ✅ 배포 준비
- requirements.txt 버전 고정
- .streamlit/config.toml 설정 추가
- STREAMLIT_DEPLOY.md 배포 가이드

### 📦 추가 기능
- Next.js + Supabase 버전 (idebate-next/) - 200+ 사용자용
- 관리자 대시보드
- 자동 평가 시스템

## 🎯 예상 효과

- **동시 접속**: 10-15명 → **20-25명**
- **토큰 비용**: **90% 절감**
- **에러율**: **대폭 감소**

## 🧪 테스트 계획

1. ✅ claude 브랜치로 Streamlit Cloud 배포
2. ⏳ 24-48시간 모니터링
3. ✅ 안정성 확인 후 이 PR 승인

---

**⚠️ 중요**: 지금은 PR만 생성하고 **머지하지 마세요!**
claude 브랜치 배포 후 24-48시간 테스트가 먼저입니다.
```

### 🖱️ 할 일:
1. **설명 칸**을 클릭
2. 위 설명을 **전체 복사해서 붙여넣기**

## 🆘 문제 해결

### Q: "Unable to merge" 메시지가 나옵니다
A: 괜찮습니다. 그래도 PR은 생성됩니다. "Create pull request" 버튼은 여전히 클릭 가능합니다.

### Q: 제목/설명을 잘못 입력했습니다
A: PR 생성 후에도 수정 가능합니다. PR 페이지에서 "Edit" 버튼을 누르세요.

### Q: 브랜치 이름이 너무 길어서 안 보입니다
A: 정상입니다. `claude/analyze-unknown...`으로 표시되면 맞습니다.
